# Git Básico – 1: Introducción a Git

#git #basico #fundamentos #control-de-versiones

---

## 1. ¿Qué es Git?

Git es un **sistema de control de versiones distribuido** que permite registrar, comparar y recuperar cambios realizados en archivos a lo largo del tiempo.

- Qué es: una herramienta para gestionar versiones de un proyecto
- Qué representa dentro de Git: el historial completo y confiable del código
- En qué se usa: desarrollo de software individual y en equipo

---

## 2. ¿Para qué sirve Git?

Git sirve para:

- Guardar el historial de cambios de un proyecto
- Volver a versiones anteriores sin perder trabajo
- Trabajar en paralelo mediante ramas
- Colaborar con otros desarrolladores de forma ordenada

---

## 3. ¿Cómo funciona Git internamente?

Git funciona mediante **snapshots** (instantáneas):

- Cada commit guarda el estado completo del proyecto
- Los archivos se identifican por hashes (SHA-1)
- El repositorio contiene toda la historia localmente

Estados principales de los archivos:

- Working Directory
- Staging Area (Index)
- Repository

---

## 4. Conceptos relacionados

## Repositorio (Repository)

Es el contenedor del proyecto gestionado por Git.

- Puede ser local o remoto
- Contiene archivos, historial y configuración
- Se crea con `git init` o al clonar un proyecto

Conceptualmente: es la “memoria” completa del proyecto.

---

## Commit

Un commit es una captura del estado del proyecto en un momento específico.

- Representa un cambio lógico y coherente
- Incluye un mensaje descriptivo
- Cada commit tiene un identificador único (hash)

Conceptualmente: un punto de guardado seguro.

---

## Rama (Branch)

Una rama es una línea de desarrollo independiente.

- Permite trabajar sin afectar el código principal
- La rama principal suele llamarse `main`
- Crear ramas fomenta experimentación segura

Conceptualmente: universos paralelos del proyecto.

---

## HEAD

HEAD es un puntero que indica dónde estás trabajando actualmente.

- Normalmente apunta al último commit de una rama
- Cambia cuando te movés entre ramas o commits

Conceptualmente: tu posición actual en el tiempo del proyecto.

---

## Staging Area (Index)

Es una zona intermedia entre tus archivos y el commit.

- Define qué cambios se incluirán en el próximo commit
- Se gestiona con `git add`

Conceptualmente: la mesa donde preparás lo que vas a guardar.
---

## 5. Aclaraciones importantes

- Git no es GitHub
- Git funciona offline
- Git no guarda archivos duplicados innecesariamente

---

## 6. Ejemplo conceptual

Imagina un videojuego:

- Cada **save** es un commit
- Puedes volver a un save anterior
- Puedes probar caminos alternativos sin romper la partida principal

---

## 7. Ventajas de Git

- Control total del historial
- Trabajo seguro y reversible
- Escalabilidad
- Uso estándar en la industria

---

## 8. Buenas prácticas desde el inicio

- Commits pequeños y claros
- Mensajes descriptivos
- Entender antes de automatizar

---

## 9. Notas importantes

- Git piensa en estados, no en archivos sueltos
- Aprender Git es aprender a pensar en versiones

---

## 10. Conceptos clave a recordar

- Control de versiones
- Snapshot
- Commit
- Historial

---

## 11. Resumen final

Git es la base para trabajar de forma profesional con código. Dominar sus fundamentos es esencial antes de avanzar a ramas, trabajo remoto y flujos avanzados.
