# Qué es una rama (branch) en Git

## 1. Definición

Una **rama (branch)** en Git es una línea independiente de desarrollo que permite trabajar sobre cambios sin afectar directamente al código principal del proyecto.

Cada rama representa una versión alternativa del historial del repositorio, que puede evolucionar de forma paralela a otras ramas.

---

## 2. Propósito

Las ramas se utilizan para:

- Desarrollar nuevas funcionalidades sin romper la versión estable.
- Corregir errores de forma aislada.
- Experimentar con cambios sin riesgo.
- Organizar el trabajo en equipo.
- Mantener un flujo de trabajo ordenado y seguro.

---

## 3. Funcionamiento interno

Internamente, una rama es **un puntero a un commit específico**.

- Git no copia archivos al crear una rama.
- Solo crea un nuevo puntero al commit actual.
- A medida que se hacen commits en una rama, ese puntero avanza.

Git utiliza un puntero especial llamado **HEAD**, que indica:
- En qué rama estás
- Sobre qué commit estás trabajando actualmente

---

## 4. Conceptos relacionados

- **Commit**: estado guardado del proyecto.
- **HEAD**: referencia al commit actual.
- **main / master**: rama principal del proyecto.
- **merge**: unión de ramas.
- **checkout / switch**: cambio entre ramas.

---

## 5. Aclaraciones importantes

- Crear una rama **no modifica** el código existente.
- Cambiar de rama puede alterar los archivos del directorio de trabajo.
- El historial de commits puede divergir entre ramas.
- Las ramas son livianas y rápidas en Git.

---

## 6. Ejemplo conceptual

Imaginemos este escenario:

- `main`: versión estable del proyecto.
- Se crea una rama `feature-login`.
- En `feature-login` se agregan commits para una nueva funcionalidad.
- `main` continúa intacta.
- Al finalizar, `feature-login` se puede fusionar con `main`.

---

## 7. Ventajas de usar ramas

- Aislamiento de cambios.
- Menor riesgo de errores.
- Trabajo paralelo eficiente.
- Historial claro y organizado.
- Facilita la colaboración en equipo.

---

## 8. Buenas prácticas

- Usar nombres descriptivos para las ramas.
- Mantener `main` siempre estable.
- Crear ramas para cada funcionalidad o fix.
- Eliminar ramas que ya no se usan.
- Hacer commits pequeños y claros en cada rama.

---

## 9. Notas adicionales

- Git permite crear tantas ramas como sea necesario.
- Las ramas son locales por defecto.
- Pueden existir ramas locales y remotas.
- Entender ramas es clave para dominar Git.

---

## 10. Conceptos clave

- Rama
- Commit
- HEAD
- main
- Historial
- Merge

---

## 11. Resumen final

Una rama en Git es una herramienta fundamental que permite desarrollar de forma paralela, segura y ordenada.  
Comprender cómo funcionan las ramas y el puntero HEAD es esencial para trabajar profesionalmente con Git, tanto en proyectos individuales como en equipos.
