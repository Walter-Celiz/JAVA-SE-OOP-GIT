## 2) Teoría

### ¿Qué es Java?

Java es un lenguaje de programación creado por **Sun Microsystems** y actualmente mantenido por **Oracle**.  

Fue diseñado con tres objetivos fundamentales:

- Simplicidad y robustez
- Programación orientada a objetos
- Independencia del sistema operativo

Su filosofía principal es:

> **Write Once, Run Anywhere**  
> Escribe una vez, ejecuta en cualquier lugar


### ¿Cómo funciona Java?

Java no se ejecuta directamente sobre el sistema operativo. Su funcionamiento se basa en un proceso intermedio:

1. El programador escribe código fuente en archivos `.java`
2. El compilador de Java convierte ese código en **bytecode**
3. El bytecode es ejecutado por la **JVM (Java Virtual Machine)**

Gracias a este modelo, el mismo programa puede ejecutarse en distintos sistemas operativos sin modificar el código.


### La JVM (Java Virtual Machine)

La JVM es un componente fundamental de Java.  

Sus responsabilidades principales son:

- Ejecutar el bytecode
- Gestionar la memoria
- Manejar errores en tiempo de ejecución
- Garantizar la portabilidad del programa

Cada sistema operativo tiene su propia implementación de JVM, pero el bytecode es el mismo para todos.


### Características principales de Java

Java se caracteriza por:

- Ser **orientado a objetos**
- Tener tipado **fuerte**
- Gestionar la memoria automáticamente (Garbage Collector)
- Ser multiplataforma
- Tener una sintaxis clara y estructurada

Es ampliamente utilizado en:

- Aplicaciones empresariales
- Desarrollo backend
- Android
- Sistemas complejos y videojuegos


### ¿Por qué Java para este proyecto?

Java es ideal para este proyecto porque:

- Permite modelar entidades complejas mediante clases y objetos
- Facilita el aprendizaje de conceptos de OOP
- Es un lenguaje usado en entornos profesionales reales
- Escala desde ejemplos simples hasta sistemas grandes

## 3) Integración conceptual con el RPG

En el proyecto RPG:

- Los personajes serán representados como **clases**
- Las habilidades y acciones serán **métodos**
- El estado del juego se manejará con **objetos**
- El flujo principal del juego partirá de un punto de entrada claro

Java permite representar este tipo de lógica de forma clara y estructurada.
