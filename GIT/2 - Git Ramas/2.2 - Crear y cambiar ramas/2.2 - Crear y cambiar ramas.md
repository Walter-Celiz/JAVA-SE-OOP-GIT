## 1. Definición

Crear y cambiar ramas en Git consiste en generar nuevas líneas de desarrollo (ramas) y moverse entre ellas para trabajar en funcionalidades, pruebas o cambios sin afectar directamente a la rama principal del proyecto.

## 2. Propósito

El propósito de usar ramas es permitir un desarrollo **paralelo, ordenado y seguro**, manteniendo la estabilidad de la rama principal (`main`) mientras se desarrollan nuevas ideas o funcionalidades.

## 3. Funcionamiento interno

Git maneja las ramas como **punteros a commits**.

- Al crear una rama, Git crea un nuevo puntero al commit actual.
    
- Al cambiar de rama, Git mueve el puntero `HEAD`.
    
- El contenido del directorio de trabajo se actualiza para reflejar el estado de la rama activa.
    

No se duplican archivos, lo que hace que las ramas sean livianas y rápidas.

## 4. Conceptos relacionados

- `HEAD`
    
- Commit
    
- Rama actual
    
- Rama principal (`main`)
    
- Historial de commits
    
- Merge (fusión de ramas)
    

## 5. Aclaraciones importantes

- Crear una rama **no implica cambiar automáticamente** a ella.
    
- No se puede borrar una rama que esté en uso.
    
- Cambiar de rama puede modificar archivos del directorio de trabajo.
    
- Las ramas locales y remotas son independientes.
    

## 6. Ejemplo conceptual

Podemos pensar el historial del proyecto como un camino principal (`main`).

Crear una rama es como tomar un desvío:

- Se trabaja de forma aislada.
    
- Se realizan cambios sin afectar el camino principal.
    
- Al finalizar, se puede volver y fusionar los cambios.
    

## 7. Ventajas

- Desarrollo aislado por funcionalidad
    
- Menor riesgo de errores en la rama principal
    
- Facilita el trabajo en equipo
    
- Permite experimentar sin consecuencias
    

## 8. Buenas prácticas

- Usar nombres claros y descriptivos para las ramas
    
- Crear una rama por tarea o funcionalidad
    
- Cambiar de rama con el directorio limpio
    
- Eliminar ramas que ya fueron fusionadas
    

## 9. Notas

- Git permite crear y cambiar de rama en un solo comando.
    
- Las ramas son una de las características más potentes de Git.
    
- Un buen uso de ramas mejora notablemente el flujo de trabajo.
    

## 10. Conceptos clave

- Rama
    
- `HEAD`
    
- Crear rama
    
- Cambiar rama
    
- Rama activa
    

## 11. Resumen final

Las ramas permiten organizar el desarrollo de un proyecto de manera profesional.  
Gracias a ellas, es posible trabajar en paralelo, probar ideas y mantener la estabilidad del código principal.
