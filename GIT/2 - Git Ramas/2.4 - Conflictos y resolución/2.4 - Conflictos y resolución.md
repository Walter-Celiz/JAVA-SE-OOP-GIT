## ¿Qué es un conflicto en Git?

Un **conflicto** ocurre cuando Git no puede decidir automáticamente cómo combinar cambios, generalmente durante un **merge** o **rebase**, porque **dos ramas modificaron la misma parte de un archivo de forma diferente**.

Git se detiene y requiere **intervención humana** para resolver el problema.

---

## ¿Cuándo aparecen conflictos?

Los conflictos suelen aparecer cuando:

- Dos ramas modifican **la misma línea** de un archivo
- Una rama **borra** un archivo que otra **modifica**
- Se hace un merge sin sincronizar cambios recientes
- Se trabaja en equipo sin coordinar responsabilidades

---

## ¿Qué hace Git ante un conflicto?

1. Detiene el merge
2. Marca los archivos en conflicto
3. Inserta **marcadores especiales** dentro del archivo
4. Espera a que el usuario los resuelva manualmente
5. Requiere confirmar la resolución con un commit

---

## Conceptos clave

- Conflicto
- Marcadores de conflicto
- Resolución manual
- Estado `MERGING`
- Commit de resolución

---

## Flujo general de resolución

1. Provocar o detectar el conflicto
2. Analizar los marcadores
3. Editar el archivo correctamente
4. Marcar el conflicto como resuelto
5. Confirmar con un commit

---

## Nota importante

Los conflictos **no son errores**, son una parte normal del trabajo con Git y del trabajo en equipo.

Saber resolverlos correctamente es una **habilidad fundamental**.
