## 1) Descripción

En esta unidad se aprende cómo Java transforma el código fuente (`.java`) en un programa ejecutable y cómo se ejecuta desde la línea de comandos.  
Se comprende el rol del **JDK**, la **JVM** y la diferencia entre **compilar** y **ejecutar** un programa Java.

Este conocimiento es clave para:
- Distinguir errores de compilación y de ejecución  
- Trabajar sin depender de un IDE  
- Entender qué ocurre internamente al ejecutar un programa Java

---

## 2) Teoría

### Archivos involucrados

- **Archivo fuente (`.java`)**  
  Contiene el código escrito por el programador.

- **Archivo bytecode (`.class`)**  
  Resultado de la compilación, independiente del sistema operativo.

---

### Proceso de compilación

La compilación convierte el archivo `.java` en bytecode `.class`.

Comando utilizado: `javac Compilacion.java`

- Si no hay errores, se genera el archivo `.class`
- Si hay errores de sintaxis, la compilación falla y no se genera el bytecode

---

### Proceso de ejecución

La ejecución consiste en que la JVM interprete y ejecute el bytecode.

Comando utilizado: `java Compilacion`


Notas importantes:
- No se escribe la extensión `.class`
- La JVM busca el método `main`

---

### Rol de la JVM

La JVM se encarga de:
- Cargar el bytecode
- Verificarlo
- Interpretarlo o compilarlo dinámicamente
- Ejecutar el programa

Esto permite que Java cumpla con el principio **Write Once, Run Anywhere**.

---

### Compilación vs Ejecución

| Compilación | Ejecución |
|------------|----------|
| Usa `javac` | Usa `java` |
| Genera `.class` | Ejecuta el programa |
| Detecta errores de sintaxis | Detecta errores en tiempo de ejecución |
| Se realiza una vez | Puede repetirse múltiples veces |


---

## 3) Integración RPG

En el proyecto RPG:
- Cada clase del juego se compila a bytecode
- La JVM ejecuta todas las clases necesarias
- Un error de compilación impide que el juego funcione
- Un error de ejecución puede romper una mecánica del juego

## 5) Notas adicionales

- Java no ejecuta archivos `.java` directamente
- Sin archivo `.class` no hay ejecución
- El nombre del archivo debe coincidir con la clase `public`
- Los IDE automatizan estos procesos, pero no los reemplazan
