## 1) Descripción  

En este tema aprenderás a utilizar la clase `Scanner` para capturar datos ingresados por el usuario desde la consola.  
  
Esto es fundamental porque:  
- Permite crear programas interactivos  
- Introduce decisiones dinámicas en tiempo de ejecución  
- Es clave para el desarrollo del RPG (elección de acciones, nombres, decisiones del jugador)  
  
---  
  
## 2) Teoría  
  
### ¿Qué es Scanner?  

`Scanner` es una clase de Java que permite leer datos desde una fuente de entrada, generalmente el teclado (`System.in`).  
  
---  
  
### Importación  

Para usar `Scanner`, es obligatorio importar:

```java
import java.util.Scanner;
```

---

### Creación del objeto  

```java
Scanner scanner = new Scanner(System.in);
```

---

### Métodos principales  

| Método        | Descripción                          | Ejemplo de entrada |
|--------------|--------------------------------------|--------------------|
| nextInt()    | Lee un número entero                 | 10                 |
| nextDouble() | Lee un número decimal                | 3.14               |
| nextLine()   | Lee una línea completa de texto      | Hola mundo         |
| next()       | Lee una sola palabra                 | Hola               |

---

### Ejemplo básico  

```java
import java.util.Scanner;

public class ScannerBasico {  
	public static void main(String[] args) {  
		Scanner scanner = new Scanner(System.in);

		System.out.print("Ingresa tu nombre: ");  
		String nombre = scanner.nextLine();  
  
		System.out.print("Ingresa tu edad: ");  
		int edad = scanner.nextInt();  
  
		System.out.println("Hola " + nombre + ", tienes " + edad + " años.");  
	}
}
```


---

### Problema común: buffer  

Cuando se combina `nextInt()` con `nextLine()`, puede producirse un salto inesperado:

```java
int numero = scanner.nextInt(); 
 
scanner.nextLine(); // limpiar buffer  
String texto = scanner.nextLine();
```

Regla importante:  
Siempre limpiar el buffer cuando cambies de lectura numérica a texto.

---  
  
## 4) Notas adicionales  
  
### Buenas prácticas  

- Crear un solo `Scanner` por programa  
- Cerrar el Scanner al final si es necesario  
- Validar siempre la entrada del usuario  
  
---  
  
### Errores comunes

- No limpiar buffer  
- Mezclar `nextLine()` con `nextInt()` sin control  
- No validar opciones del usuario  
  
---  
  
### Tip profesional  

Centralizar la entrada de datos en métodos:

```java
public static int leerOpcion(Scanner scanner) {  
System.out.print("Elige una opción: ");  
return scanner.nextInt();  
}
```

Esto mejora la organización del código.

## Resumen  
  
`Scanner` permite capturar datos del usuario y hacer programas interactivos.
