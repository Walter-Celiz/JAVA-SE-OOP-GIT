## 1. Descripción

En este tema aprenderemos a instalar Git en distintos sistemas operativos, configurarlo por primera vez y verificar que la instalación sea correcta. Además, configuraremos nuestro nombre y correo electrónico global, elegiremos un editor de texto predeterminado y revisaremos la configuración actual de Git.

## 2. Teoría

### Instalación de Git

- Git se puede instalar en **Windows, macOS y Linux**.
    
- Cada sistema tiene su propio instalador o gestor de paquetes:
    
    - **Windows**: instalador desde [git-scm.com](https://git-scm.com/downloads) o mediante `winget`.
        
    - **macOS**: `brew install git` o instalador oficial.
        
    - **Linux**: gestor de paquetes del sistema (`apt`, `yum`, `dnf`, etc.).
        

### Configuración inicial

- Es **fundamental** definir el nombre y correo electrónico global:
    
    `git config --global user.name "Tu Nombre" git config --global user.email "tuemail@dominio.com"`
    
- Esto identifica tus commits en todos los repositorios locales.
    
- Para configuraciones **solo de un repositorio**, se puede usar `--local` en lugar de `--global`.
    

### Editor de texto predeterminado

- Git permite elegir un editor para los mensajes de commit:
    
    `git config --global core.editor "nombre_editor"`
    
- Ejemplos: `vim`, `nano`, `code --wait` (VSCode).
    
- Elegir un editor familiar evita problemas al escribir mensajes de commit.
    

### Verificación de configuración

- Comandos importantes:
    
    `git config --list git --version`
    
- Permiten revisar la configuración actual y la versión instalada.
    

### Buenas prácticas

- Siempre verificar la instalación antes de empezar un proyecto.
    
- Mantener la configuración global consistente para todos tus repositorios.
    
- Evitar cambios frecuentes de nombre o email para no generar inconsistencias en los commits.
