## 1. Definición  

**Git Stash** es una funcionalidad que permite guardar temporalmente cambios no confirmados (working directory y staging area) sin necesidad de crear un commit, dejando el repositorio en un estado limpio para cambiar de rama o realizar otra tarea.

---

## 2. Propósito  

El propósito de `stash` es **interrumpir el trabajo actual de forma segura**, permitiendo:
- Cambiar de rama rápidamente.
- Actualizar el repositorio sin perder cambios locales.
- Posponer trabajo incompleto sin ensuciar el historial de commits.

---

## 3. Funcionamiento interno  

Cuando se ejecuta `git stash`:
- Git guarda los cambios actuales en una pila interna (stack).
- El working directory vuelve al último commit confirmado.
- Cada stash se guarda como una entrada independiente, identificada por un índice (`stash@{0}`, `stash@{1}`, etc.).

Internamente, Git crea commits especiales que no forman parte del historial normal del proyecto.

---

## 4. Conceptos relacionados

- **Working Directory**: área de trabajo con archivos modificados.
- **Staging Area**: área de preparación (`git add`).
- **HEAD**: referencia al último commit actual.
- **Ramas (branches)**: contexto donde se aplican los cambios del stash.
- **Commit**: confirmación permanente (a diferencia del stash, que es temporal).

---

## 5. Aclaraciones importantes  

- El stash **no reemplaza commits**.
- Los cambios stasheados **no se comparten** con el repositorio remoto.
- Por defecto, `git stash` guarda archivos modificados y en staging, pero **no archivos no trackeados** (a menos que se indique explícitamente).

---

## 6. Ejemplo conceptual  

Situación:
- Estás trabajando en una feature.
- Surge una urgencia para cambiar de rama.
- Tienes archivos modificados sin terminar.

Solución:
- Guardas los cambios con `stash`.
- Cambias de rama.
- Luego recuperas el trabajo exactamente donde lo dejaste.

---

## 7. Ventajas  

- Evita commits innecesarios o incompletos.
- Permite cambiar de contexto rápidamente.
- Mantiene el historial limpio.
- Ideal para trabajo diario y correcciones urgentes.

---

## 8. Buenas prácticas  

- Usar stash solo para **cambios temporales**.
- Aplicar y limpiar stashes antiguos regularmente.
- Nombrar los stashes cuando sea posible.
- No acumular demasiados stashes sin control.

---

## 9. Notas  

- El stash funciona como una **pila (LIFO)**.
- Se puede aplicar un stash en una rama distinta a la que se creó.
- Pueden surgir conflictos al aplicar un stash, igual que en un merge.

---

## 10. Conceptos clave 

- Stash
- Working Directory
- Staging Area
- Stack
- Aplicar cambios
- Eliminar stash

---

## 11. Resumen final  

`git stash` es una herramienta esencial para el trabajo profesional con Git, ya que permite pausar tareas en progreso de forma segura y flexible, facilitando el cambio de contexto sin perder información ni afectar el historial del proyecto.
