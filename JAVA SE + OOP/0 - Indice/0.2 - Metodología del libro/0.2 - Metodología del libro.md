# Metodología del Libro y del Proyecto RPG

Este libro sigue un método de aprendizaje incremental llamado Estilo A.  
Cada capítulo introduce un concepto de Java y, al mismo tiempo, incorpora ese conocimiento al Proyecto RPG que se desarrollará a lo largo de todo el curso.

---

## 1. Objetivos generales del libro

- Comprender Java SE desde cero con una base sólida.
- Aprender Programación Orientada a Objetos de forma progresiva.
- Desarrollar un proyecto práctico tipo RPG simple que crece capítulo a capítulo.
- Integrar buenas prácticas de documentación utilizando Obsidian.
- Registrar el avance mediante Git para reforzar hábitos profesionales.

---

## 2. Cómo funciona 

El aprendizaje se divide en dos caminos que avanzan en paralelo:

1. El camino teórico: capítulos del libro.
2. El camino práctico: el Proyecto RPG.

Cada capítulo incluye:

1. Objetivo del capítulo  
2. Explicación clara del tema  
3. Ejemplos de código  
4. Ejercicio práctico  
5. Avance del Proyecto RPG  
6. Resumen

El progreso del juego se vincula directamente con la teoría aprendida en cada etapa.

---

## 3. Relación entre los capítulos y el Proyecto RPG

El desarrollo del RPG está distribuido en etapas:

### Parte I – Fundamentos
- Estadísticas básicas del personaje
- Operaciones simples de vida, ataque y defensa
- Menú básico en consola
- Primer archivo simple con datos

### Parte II – Intermedio
- Clases para personajes, enemigos e items
- Inventario con colecciones
- Sistema de turnos simple
- Manejo de excepciones
- Carga y lectura de archivos
- Uso de BigInteger y BigDecimal para economía
- JavaTime para tiempos, cooldowns o eventos diarios

### Parte III – OOP
- Herencia aplicada a tipos de personajes
- Polimorfismo en acciones de combate
- Interfaces para habilidades especiales
- Clases abstractas para entidades del juego
- Lambdas para comportamientos
- Streams para registros del combate

### Parte IV – Avanzado
- Organización del proyecto en paquetes
- Inventarios con Generics
- Concurrencia básica
- Serialización de la partida
- NIO2 para un manejo de archivos moderno
- Optional para estados internos del juego

### Parte V – Proyecto final
- Integración completa del RPG
- Persistencia avanzada
- Optimización del código
- Menú final del juego
- Ejecución y publicación del proyecto

---

## 4. Uso de Git en el aprendizaje

Cada capítulo producirá al menos tres commits:

1. Contenido teórico escrito
2. Ejercicio del capítulo
3. Avance concreto del RPG

Esto crea un historial de aprendizaje claro y ordenado.

Es recomendable utilizar dos repositorios:

- `java-book` para las notas y documentación
- `java-rpg` para el proyecto del juego

---

## 5. Cómo utilizar este libro dentro de Obsidian

1. Navegar los capítulos usando la estructura de carpetas.
2. Mantener un capítulo abierto junto con el IDE para practicar.
3. Actualizar las notas con ejemplos adicionales si es necesario.
4. Seguir los ejercicios y realizar commits al completar cada parte.
5. Integrar progresivamente lo aprendido en el Proyecto RPG.

---

## 6. Flujo de estudio recomendado

1. Leer el capítulo completo.
2. Entender los ejemplos y escribirlos por tu cuenta.
3. Resolver el ejercicio práctico.
4. Programar la parte del RPG del capítulo.
5. Hacer commits en Git.
6. Continuar al siguiente capítulo.
