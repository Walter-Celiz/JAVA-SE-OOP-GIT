## 1. Definición

El **rebase avanzado** es un conjunto de técnicas que amplían el uso básico de `git rebase`, permitiendo **editar, reordenar, combinar (squash) o eliminar commits** existentes.  
Se utiliza principalmente para **limpiar y optimizar el historial de commits** antes de integrarlos a una rama principal.

---

## 2. Propósito

El objetivo del rebase avanzado es:

- Mantener un **historial de commits claro y legible**
- Agrupar cambios relacionados en commits coherentes
- Corregir errores en commits ya creados
- Preparar ramas antes de un merge o pull request
- Aplicar buenas prácticas en equipos profesionales

---

## 3. Funcionamiento interno

Git rebase avanzado funciona recreando commits uno por uno sobre una nueva base, permitiendo intervenir en cada paso.

Proceso general:
1. Git identifica los commits a modificar
2. Detiene la ejecución en cada commit (si se solicita)
3. Aplica acciones definidas por el usuario (editar, squash, reordenar)
4. Genera nuevos commits con nuevos hashes

Importante: **los commits originales dejan de existir**, ya que se crean nuevos.

---

## 5. Aclaraciones importantes

- Nunca se debe usar rebase avanzado sobre commits **ya compartidos** con otros colaboradores
- Modificar el historial cambia los hashes de los commits
- Es una herramienta poderosa pero peligrosa si se usa sin cuidado
- Ideal para ramas locales de feature o experimentación

---

## 6. Ejemplo conceptual

Un historial inicial:

A --- B --- C --- D

Con rebase avanzado se puede transformar en:

A --- B --- E

Donde:
- C y D fueron combinados (squash)
- E es un commit nuevo más limpio y descriptivo

---

## 7. Ventajas

- Historial más profesional y legible
- Commits mejor definidos
- Facilita revisiones de código
- Reduce commits innecesarios o ruidosos

---

## 8. Buenas prácticas

- Usar rebase avanzado **antes** de hacer push
- Revisar cuidadosamente los commits antes de confirmar el rebase
- Hacer backup de la rama si el cambio es complejo
- Preferir mensajes de commit claros tras squash o edición

---

## 9. Notas

- Rebase avanzado es una habilidad intermedia/avanzada en Git
- Es muy usado en flujos con Pull Requests
- Muchas empresas lo exigen para mantener repositorios limpios

---

## 11. Resumen final

El rebase avanzado permite **reescribir la historia de una rama de forma controlada**, mejorando la calidad del historial y facilitando el trabajo en equipo.  
Usado correctamente, es una de las herramientas más potentes de Git para desarrollo profesional.
